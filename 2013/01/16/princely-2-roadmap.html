Princely 2.0 Roadmap

16 Jan 2013

The next planned set of non-bugfix work includes some big changes to the API, so we’re going to bump the version up to 2.0.0.

  • Princely class pdf generation responsibilities will be moved to Princely::Pdf. Princely will be changed to a module and used for global princely configuration.
  • Princely::Pdf will accept a template or html string (eg Princely::Pdf.new('<html></html>').render)
  • HTML used for PDF generation will be accessible on the class.
  • PdfHelper will become Princely::PdfHelper.

I want to also keep developing the test suite. Some examples of unique behaviour that you’ve experienced would go a long way to bedding down a functional suite. Please send me an email or open a pull request.

Please leave a comment if you have any queries or concerns.
